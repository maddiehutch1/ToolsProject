# MVP Document
## Mental Health Companion — Agentic Chat Application

**Version:** 2.0 | **Date:** March 9, 2026

---

## Goal

Ship the simplest working version that demonstrates:
1. Chat web UI with persistent disclaimer (SSE streaming recommended)
2. LangGraph agent choosing between three tools: RAG search, web search, calculator
3. Responses grounded in ≥5 real knowledge base documents with source attribution
4. Multi-turn conversation memory
5. Hardcoded crisis resource response for distress signals

## Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Backend | Python 3.11+, FastAPI | Native async; `StreamingResponse` for SSE |
| Agent | LangGraph `StateGraph` | Explicit graph; native `.astream_events()` |
| LLM | `gpt-4o-mini` via `langchain-openai` | Low cost; strong tool-calling |
| RAG | LangChain loaders + FAISS (local) | No external service; `save_local`/`load_local` |
| Embeddings | `text-embedding-3-small` | Fast, cost-effective |
| Web search | `langchain-tavily` `TavilySearch` | Official LangChain integration |
| Math eval | `simpleeval` | Safe expression parsing, no `eval` |
| Frontend | Vanilla HTML/CSS/JS | No build step |
| Config | `python-dotenv` | Standard `.env` pattern |

## Definition of Done

- [ ] `GET /health` returns `{"status": "ok", "vector_store": "loaded"}`
- [ ] Ingest script builds FAISS index from ≥5 real KB documents
- [ ] `rag_search` returns relevant chunks with source names for ≥5 queries (unit tested)
- [ ] Agent response cites source document name when answering from KB
- [ ] `calculator` correctly evaluates ≥10 expressions including PHQ-9 scoring (unit tested)
- [ ] `web_search` returns structured results for ≥3 queries (tested with live key)
- [ ] Agent routes to correct tool ≥90% of a 10-query test set
- [ ] Multi-turn follow-up returns contextually correct answer
- [ ] Crisis keyword triggers hardcoded response — LLM never invoked
- [ ] Chat UI functional in browser; streaming works; tool badges display
- [ ] Crisis card renders with distinct styling and resource links
- [ ] Disclaimer banner visible at all times
- [ ] No API keys in version control
- [ ] `README.md` has setup, ingest, and run instructions
- [ ] Code passes `ruff` or `flake8`

## Risks

| Risk | Mitigation |
|------|-----------|
| LangGraph streaming API changes | Pin version; test `.astream_events(version="v2")` in isolation first |
| RAG retrieval quality too low | Test retrieval in Phase 2 before wiring to agent |
| Crisis detection false positives | Start with a small, high-confidence keyword list |
| FAISS ingest failure | Run `ingest.py` early; verify files exist before starting server |
| Tavily quota exceeded | Mock in tests; use live key only for E2E |
