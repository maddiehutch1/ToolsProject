## 1. MVP Goal

Ship the simplest working version of the mental health companion that demonstrates:
1. A **chat web UI** in the browser with a persistent disclaimer (streaming recommended)
2. A LangGraph agent that can choose between three tools: **RAG search**, **web search**, and **calculator**
3. Responses grounded in a curated knowledge base of **at least 5 real documents**, with **source attribution** in every RAG-based reply
4. **Multi-turn conversation memory** so follow-up questions work correctly
5. A hardcoded crisis resource response for distress signals

Everything beyond this core loop is deferred to a later iteration.

## 3. Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| **Backend language** | Python 3.11+ | Strong LLM/LangChain ecosystem; familiar for coursework |
| **Web framework** | FastAPI | Native async/streaming; easy SSE with `StreamingResponse` |
| **Agent orchestration** | LangGraph `StateGraph` | Explicit, debuggable agent loop; native streaming via `.astream_events()` |
| **LLM integration** | LangChain (`langchain-openai`) | Tool-calling wrappers; integrates directly with LangGraph |
| **LLM model** | `gpt-4o-mini` | Low cost, fast, strong tool-calling support |
| **RAG pipeline** | LangChain document loaders + `RecursiveCharacterTextSplitter` | Best-in-class text ingestion and chunking |
| **Embeddings** | `text-embedding-3-small` (OpenAI) | Fast, cost-effective semantic embeddings |
| **Vector store** | FAISS (local, persisted as flat files) | No external service or database engine; simple `save_local` / `load_local` file pair |
| **Math evaluation** | `simpleeval` | Safe expression parsing without raw `eval` |
| **Web search** | `tavily-python` | Official Tavily client; simple query interface |
| **Frontend** | Vanilla HTML + CSS + JavaScript | No build step; easy to read and grade |
| **Streaming protocol** | Server-Sent Events (SSE) | Browser-native; one-directional stream |
| **Environment config** | `python-dotenv` | Standard `.env` file for API keys |

## 6. Key Implementation Details

### 6.1 LangGraph Agent (sketch)

```python
# backend/agent.py
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode
from langchain_openai import ChatOpenAI
from typing import TypedDict, Annotated
from langchain_core.messages import BaseMessage
import operator

class AgentState(TypedDict):
    messages: Annotated[list[BaseMessage], operator.add]

llm = ChatOpenAI(model="gpt-4o-mini", streaming=True)
tools = [rag_search, web_search, calculator]
llm_with_tools = llm.bind_tools(tools)

def call_model(state: AgentState):
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

def should_continue(state: AgentState):
    last = state["messages"][-1]
    return "tools" if last.tool_calls else END

tool_node = ToolNode(tools)

graph = StateGraph(AgentState)
graph.add_node("call_model", call_model)
graph.add_node("tools", tool_node)
graph.set_entry_point("call_model")
graph.add_conditional_edges("call_model", should_continue)
graph.add_edge("tools", "call_model")
agent = graph.compile()
```

### 6.2 LangGraph Streaming via `.astream_events()` (sketch)

```python
# backend/agent.py
async def run_agent_stream(messages: list, session_id: str):
    state = {"messages": messages}
    async for event in agent.astream_events(state, version="v2"):
        kind = event["event"]
        if kind == "on_chat_model_stream":
            chunk = event["data"]["chunk"].content
            if chunk:
                yield json.dumps({"type": "token", "content": chunk})
        elif kind == "on_tool_start":
            yield json.dumps({"type": "tool_use", "tool": event["name"]})
        elif kind == "on_tool_end":
            yield json.dumps({"type": "tool_done", "tool": event["name"]})
    yield json.dumps({"type": "done"})
```

### 6.4 RAG Tool with Source Citation (sketch)

```python
# backend/tools/rag_search.py
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings
from langchain_core.tools import tool

vectorstore = FAISS.load_local(
    "faiss_index",
    OpenAIEmbeddings(model="text-embedding-3-small"),
    allow_dangerous_deserialization=True,
)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

@tool
def rag_search(query: str) -> str:
    """Searches the mental health knowledge base for evidence-based techniques."""
    docs = retriever.invoke(query)
    results = []
    for doc in docs:
        source = doc.metadata.get("source", "unknown")
        results.append(f"[{source}]\n{doc.page_content}")
    return "\n\n---\n\n".join(results)
```

### 6.5 Frontend SSE Consumer (sketch)

```javascript
// frontend/app.js
async function sendMessage(userText) {
  const response = await fetch('/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ message: userText, session_id: sessionId }),
  });
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  const bubble = createAgentBubble();

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    for (const line of decoder.decode(value).split('\n')) {
      if (!line.startsWith('data: ')) continue;
      const event = JSON.parse(line.slice(6));
      if (event.type === 'token')    bubble.appendText(event.content);
      if (event.type === 'tool_use') bubble.addToolBadge(event.tool);
      if (event.type === 'crisis')   bubble.renderCrisisCard(event.content);
      if (event.type === 'done')     return;
    }
  }
}
```

## 9. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| LangGraph streaming API changes between versions | Medium | High | Pin `langgraph` version in `requirements.txt`; test `.astream_events()` in isolation first |
| Knowledge base documents too sparse for meaningful retrieval | Medium | High | Test RAG retrieval quality early in Phase 2 before building the agent |
| Crisis detection false positives interrupting normal conversation | Medium | Medium | Start with a small, high-confidence keyword list; tune before expanding |
| FAISS ingest fails or index files are corrupted | Low | Medium | Test ingest script early; verify `faiss_index/` files exist before starting the server |
| Tavily API key unavailable or quota exceeded | Low | Medium | Test with mocked responses early; keep a mock fallback in tests |
| `simpleeval` missing needed math functions | Low | Low | Pre-test supported functions before writing test cases |
| Scope creep from mental health domain complexity | Medium | High | Strictly defer edge cases and expanded knowledge base to post-MVP |
